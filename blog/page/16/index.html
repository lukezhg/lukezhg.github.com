使用Octopress发布Github Page博客

Dec 27th, 2012 | Comments

本来想尝试使用其他的文本编辑器，以后再说吧。我就用这个editplus干活了，先把使用github以及octopress搭建博客的整个过程梳理下来。

搭建的步骤如下：
1.首先是准备工作，下载和安装必要的软件。
2.然后是克隆octopress并安装
3.接下来是将octopress配置好并发布博文。
具体步骤可以重点参考这篇文章。

具体步骤纪要：

  1. 准备工作，下载gitinstaller, ruby197, devkit, 然后安装。 gitinstalller和ruby197只要一路next过去就可以安装好，ruby我选择的都是c盘根目录。devkit解压到c盘根目录。安装如下： 
    $ cd C:\DevKit

    $ ruby dk.rb init
    $ ruby dk.rb instal

布置环境：

$ gem sources --remove http://rubygems.org/  
  $ gem sources -a http://ruby.taobao.org/  
  $ gem install rdoc bundler   
  $ vi Gemfile  #将行 ：source "http://rubygems.org/" 改为:source "http://ruby.taobao.org/"
  $ bundle install  #如果出错，改执行：bundle update; rake install
  1. 克隆远程octopress到本地文件夹，本地文件夹名称无所谓，比如我的就放在G:/github/myblog
    接下来进入git shell, 在命令行中执行一系列的命令来安装octopress，命令代码如下： 
    $ git clone git://github.com/imathis/octopress.git myblog

    $ cd myblog
    $ rake setup_github_pages
    按照提示输入：git@github.com:username/username.github.com.git, 比如我的github用户名为lukezhg，那么我就需要输入 git@github.com:lukezhg/lukezhg.github.com.git

    $ rake generate   #生成本地文件

    $ rake preview #实现本地预览 访问 http://localhost:4000 查看效果 （可选） $ rake deploy #发布本地文件到服务器

保存博客源码到github source分支:

$ git add .  
  $ git commit -m 'blog source'  
  $ git push origin source
  1. 初步配置octopress
    _config.yml文件：

添加cname文件
只需要在myblog\source目录下添加一个cname的文件，并在第一行写上你的域名，保存即可。

发布新博文（post）或者新页面（page）

$ rake new_post["article name"]   
  $ rake new_page["page name"]
  1. 继续完善
    添加侧边栏内容

（1）添加导航
修改myblog\source_includes\custom目录下的navigation.html文件即可

（2）添加分类
alswl/octopress-category-list

（3）自定义网站图标
替换myblog\source目录下的favicon.png可以实现自定义网站图标

  1. 额外的说明
    （1）执行rake install如若出错： 
    $ rake install

    $ rake aborted! $ You have already activated rake 0.9.2.2, but your Gemfile requires rake 0.9.2. Using bundle exec may solve this. $ (See full trace by running task with –trace)

==》修正办法为：

$ bundle update; rake install

（2）添加CNAME文件后不生效==》手动在github网页上文件代码剪贴掉，保存，然后再粘贴回来，保存。

（3）atom.xml不生效==》使用同样的方法先把文件代码剪贴，保存，然后再粘贴回来。

如何辅助学习者在复杂网络学习环境中生存下来

Nov 29th, 2012 | Comments

当课程放在一个复杂环境中的时候，学习者很容易产生技术困扰，进而从学习进程中迷失目标方向，最后drop out。如何设计一种辅助，帮助学生克服这种困扰呢？

思路1：设计一个软件帮助学生管理自己的学习资源和进程，比如使用evernote的复制连带网址功能，就可以把相关的作业及要求摘录到笔记中，然后再使用标签进行管理即可。我觉得以后可以重点介绍一下evernote这款软件。

思路2：把复杂环境简单化，越是简单的机制越是容易理解和使用。

思路3：建立一个自己的web环境，汇总和侦听各种课程信息。比如google个人页面，wikispace个人页面等，这对学习者将提出更高的要求。

 

与懒做斗争——从回顾、梳理、总结开始

Jul 11th, 2012 | Comments

放假已经有一阵子了，但是自己一直被懒折磨着，很多问题都搁置在一边没有进展。怎么样才能摆脱这种状态呢？

在《工作就是解决问题》这本书中“先找问题的薄弱点”有这样一段内容：

当一个人在学习和工作中达到所谓的“停滞点”时，从问题的最薄弱处下手，从最容易的地方开始，先做自己擅长的、有把握的事情，往往特别有效。

所谓“停滞点”，便是到达那一点时，再用功也无法获得更多进步。这时，若是硬逼着自己冲过这个“停滞点”，便很可能产生紧张、困难等感觉。甚至形成恶性循环——越想解决问题，就越解决不了。越想闯过去，就越是原地打转。这时，你可以稍微放松一下，先做最容易的事。

而对于我而言，写作最容易的事情就是回顾、整理和总结，我曾经做了很多的笔记、书签，应该好好整理、总结一下了。

日子一天天过，事情一件件做，坚持才能有收获。

读《天眼》悬念小说所得

Jun 19th, 2012 | Comments

《天眼》这篇小说围绕着“盒子”展开，故事情节跌宕起伏，用了一天时间一口气读完。虽然并应该花这么多的时间做这样的阅读消遣，但是奈何时间已经花费了，索性把自己的收获记录一番。

读过这篇小说，我在思考一个问题，如何写好故事？特别是如何写好教育故事和学习故事？作为教师或者学习者是否也可以写出类似引人入胜的故事？我想这种教育或学习过程的记录，同样也是问题解决的过程，同样能反映教学者或者学习者的所经所感，也就有机会反映和记录真实的专业成长过程或个人学习过程。

在经历了一次次劫难之后，潇伟和赵颖终于走到一起，有情人总成眷属，但也总会有不圆满，阿雪还是最后的任务中撒手人寰。经历了这么多的事情，主人公也再重新审视人生之意义，辗转几代人，几多英雄，几多磨难，到底为何执着。最后男女主人公又共同打开了雌匣，揭起新的不可预测的故事开端，笔锋戛然而止。

人生的重大问题永远值得思考：人生意义何在？如何找到自己真正按兴趣的事情？如何把自己感兴趣的事情做到最好？

但是答案却永远在变！

成功的三大原则

Jun 19th, 2012 | Comments

智慧 努力 运气

运气原则=主体行为与环境条件恰好相符，因此人应该依照外环境条件行事

等价交换原则=主题付出努力依照固定原则交换结果，因此人应该持之以恒

智慧原则=同样的努力如果能够有智慧的选择行动方式和利用原则，可以事半功倍。

除此之外，一个人要想成功还需要勇气，改变自我和外部环境的勇气。
