如何将micro blog自动同步到国内博客

发表于 2011-12-14 更新于 2013-08-25 分类于 技术技巧

用GAE架设的micro blog非常不错，唯一的缺憾就是国内的朋友看不到，为什么看不到，我想你懂的。为了弥补这个缺憾，我就想把micro blog同步到国内的某个博客上，但似乎很难，因为国内的博客服务器也“看不到”我们appspot.com上的mblog。怎么办？于是搜索，尝试，终于找到一种暂行的方法：

首先把micro blog的RSS变为国内能访问的RSS，我选用的是Feed43（http://feed43.com/）。这步解决了，其他的就好解决了。这里有一个教程：feed43使用教程 - Unique Slacker - 博客大巴: http://kenzone.blogbus.com/logs/2295747.html

然后把RSS转换为html代码直接插入到网页上，我用的是RSS2html（http://rss.bloople.net/）。比如在博客大巴上插入自定义html就可以了。

费了吃奶的劲，终于把外面的内容转移到了里面，何苦呢！中国人总是自己个自己下套。

基于问题的学习和知识分享（PBL &　PBS）

发表于 2011-12-01 更新于 2013-01-11 分类于 openlearning

我们在教学和学习的过程中，经常会使用到基于问题的学习（problem-based Learning）策略。原因很简单，因为人们经常是为了现在解决遇到的问题而学习的，而不是为了解决未来的问题而做准备，因为问题是很难预知的。遇到了，才知道。

在基于问题的学习中，学习者可以基于自己的兴趣扩展学习，去查阅资料，却做尝试和实验，此外还要肩负起监控和调整自己学习过程的重担，元认知也能够得到锻炼。如果不能达到这些，这种教学策略对于学习成效的提高就很难起到帮助。

我们不仅可以围绕问题展开学习，同样的我们也应该围绕问题展开知识的分享。在围绕问题分享知识的时候，我们要做三件事情：

  1. 具体清晰的描述自己遇到问题和自己想达成的目标。
  2. 给出问题解决的关键思想和具体解决步骤。
  3. 提供相关的扩展阅读的资料和参考文献。
学习者完全可以只用这三个环节来分享知识，这会使得知识的交流的焦点更为集中，分享更有针对性和成效性。通常我们经常缺乏清晰描述自己问题的意识，以至于自己的解决方案究竟在遇到什么问题时才会派上用场也不清楚。还有就是只关注了给出具体的步骤，但是并没有提炼问题解决的核心思想，这样容易让读者只有步骤细节，而缺乏整体观，很容易产生误解和迷惑。此外，提供信息的参考来源有利于让读者了解信息的前前后后，或者为了进一步扩展阅读提供了可能。

会心跳的空调

发表于 2011-11-30 更新于 2013-08-25 分类于 生活

今天天气不好，一直下着小雨，雨不是很大，但是借着较强的风势，横飞乱舞，打着伞身子也会被淋湿了一半。回到家，坐在办公桌前面打开电脑，正准备做一个flash演示文稿模板，突然听到一种很奇怪的声音，嘭~啪~嘭~啪~，周而复始，有点像流水的声音，又好像似一个人的心跳，但是家里明明只有我一个人。

于是仗着胆子站起身来，细听声音的来处，原来就是侧对着办公桌的哪个空调发出的声音。好奇怪啊，这个空调明明没有开，怎么会有声音呢。外面的风刮的更厉害了，又传来了嘭啪嘭啪的声音，于是猜想是不是与这风有关系呢？本来想静下来做点事情的，这个空调老是给我执照噪音怎么行呢，于是就开始想解决的办法。

我猜想可能是风速变化很大，时快时慢，使得空调的循环管受的压力不断变化，进而造成里面的液体的震荡所以产生了这种类似“心跳”的声音。加入这种猜想成立的话，如何解决呢? 首先想到的是用一个大塑料袋子把空调的风扇整个包起来来，这样这个风扇的管道就不对外开放了，但是找了一会没找到合适的袋子，而且风扇在阳台栏杆的外面，需要越过栏杆才能将其罩住，栏杆很脏，而且有一定危险性，于是放弃了这种方法。那么怎么办呢？

仔细想了一下，由于天气有点冷，房子的门窗都关着，整个房子就好像一个秘密的容器，只有空调的循环管与外面相连，由于风速变化造成压力变化，所以会传来空调的心跳声。对了，如果我在打开一个通道不久行了嘛，于是我就把阳台的拉门打开了，果然空调的心跳停下来了，问题解决！

什么是知识共享协议（CC协议）

发表于 2011-11-28 更新于 2013-01-11 分类于 openlearning ， web2.0

在网络学习的过程中，我们经常会使用用别人的作品，或者我们自己也会制作一些作品，那么那些作品可以进行演绎创作，哪些不可以，我们使用了是否侵权，我们制作的作品又再什么样的协议下发行，这些成为迫切需要解决的问题。

目前网络上使用广泛的，大家一般共同遵循的协议就是CC协议，英文为（Creative Commons），目前大陆也有网站叫做知识共享中国大陆。

在版权声明中经常会使用一些图标，比如署名、非商业性使用、禁止演绎以及相同方式共享等图标。使用者可以组合这些图标来声明自己作品的许可协议。那么这些图标的含义是什么呢？这里摘录知识共享中国大陆网站的一段文字如下：

许可协议说明

您在自己的作品上使用知识共享许可协议，并不意味着放弃您的著作权，而是在特定的条件下将您的部分权利授予公共领域内的使用者。哪些特定的条件呢？您可以在此处看到所有知识共享许可协议及其简单的介绍。所有的许可协议都要求您以作者或者许可人的名义署名。

您可以将以下的选项进行组合、搭配，由此将构成我们的六套核心知识共享许可协议。

署名： 您允许他人对自己享有著作权的作品及演绎作品进行复制、发行、展览、表演、放映、广播或通过信息网络向公众传播，但在这些过程中对方必须保留您对原作品的署名。

例如：小红在她的摄影作品上使用了“署名”的授权条款，因为她希望在他人在使用她的图片时尊重自己的署名。小明在网上找到她的摄影作品并希望在自己网站的首页上进行展示。在这种情况下，小明可以将小红的图片放在他的网站上，只要清楚地指明小红是作者即可。

非商业性使用：您允许他人对您享有著作权的作品及演绎作品进行复制、发行、展览、表演、放映、广播或通过信息网络向公众传播，但仅限于非商业性目的。

例如：小明在他的网站上以包含“非商业性使用”要素的许可协议发表了他的摄影作品。在这种情况下，小红可以打印使用小明的这幅照片。但是，如果没有小明的允许，小红不得出售这张打印的照片。

禁止演绎：您允许他人对您的作品原封不动地进行复制、发行、展览、表演、放映、广播或通过信息网络向公众传播，但不得进行演绎创作。

例如：小红在自己的一首歌曲上使用包含“禁止演绎”要素的许可协议。小明想截取小红歌曲片段混合在他的作品中而产生出全新的歌曲。在这种情况下，没有小红的允许，小明将不能这么做（除非他的歌曲构成合理使用）。

相同方式共享：只有在他人对演绎作品使用与您的原作品相同的许可协议的情况下，您才允许他人发行其演绎作品。

注：许可协议协议不能同时包含“相同方式共享”和“禁止演绎”许可要素，“相同方式共享”要素仅适用于演绎作品。

例如：小明的网上照片采用了包含“非商业性使用”和“相同方式共享”要素的许可协议。小红是一位业余的抽象拼贴
艺术家，她截取小明的照片并且将其使用到她的一件拼贴作品之中，在上述情况下，小红必须按照小明作品所适用的许可协议的要求，在她的拼贴作品上适用包含
“非商业性使用”和“相同方式共享”要素的许可协议，将自己的作品以同于小明所选的许可协议发表。

我们的范例页面上有更多的例子。请您注意，每种许可协议除了包含上述特别许可条款外，都包含了一套共同的基本权利和限制。

采用授权条款

当您完成选择之后，您将会得到以下三种方式表述的的许可协议：

  1. 普通文本：对许可协议所作的通俗、简洁摘要，并配以相关图示。
  2. 法律文本：能确保许可协议在法院具有效力的完整协议版本。
  3. 元数据：将许可协议转换为机器可读的形式，能协助搜索引擎以及其他应用程序查找您所选择的许可协议。

使用许可协议

接着，您应该在您的网站中，作品的旁边，设置一个知识共享的“保留部分权利”的按钮。这里是如何进行设置的协助与指引。这个按钮将会链接到普通文本，这样全世界都能注意到该许可协议。如果您发现有人违反您的许可协议，您便能以此为基础提起侵害著作权的诉讼。

（以上内容来自：知识共享中国大陆 http://cn.creativecommons.org/licenses/licenses_exp/）

大脑错觉（Brain bugs）

发表于 2011-11-27 更新于 2013-08-25 分类于 book

在笑来的博客上看到，这本书《Brain bugs》，笑说有趣的很。于是打开看了一下目录还有网站的Demos如下：
Brain Bugs | Demos
: http://www.brainbugs.org/demos.html

上面的页面上列举了很多brain bugs。
比如McGurk Effect，上面有一个女人在发言，但是当你闭上眼睛的时候，你听到的是一个声音，当你睁开眼睛的时候你听到的是另外一个声音。（我猜测是收到了发音者口型的影响）

再比如Baker-baker Paradox
我们经常会忘记某个人的名字，但是多半会记得他的职业。即使是一个叫Baker的baker，我们还是容易自己的他是一个面包师。原因在于，我们记忆一个人的职业的时候，会有很多相关的概念协同起来，而名字则很少协同记忆。（也许老师们也可以尝试为学生的名字富于内涵，从而便于记忆）
这也提示我们，在教学过程中，应该把新概念与学生的已有概念产生更多的来联系，使其协同起来，而不是像名字一样孤零零的，最后被遗忘。

再如Flash Lag Effect
我在预测红球的时候，总是预测超前50左右，这就是为什么足球边裁判罚越位中有24%为误判的原因。其原因是我们的视觉系统在关注其他信号的同时，总是倾向于超前估计运动物体的位置。

再如Leaning Tower Illusion
看起来右侧的塔会更加的倾斜，但是其实两个塔的照片是完全相同。

从软件更新到新陈代谢

发表于 2011-11-27 更新于 2013-01-11 分类于 Firefox

最近有点反感软件更新了。为啥呢？主要是兼容问题。
举例：

  1. 我以前用火狐浏览器挺好的，安装了很多好用的插件，也花了好多时间和心思定制自己的火狐。有一天，看到了火狐的更新提醒，于是点击了更新，结果原有的很多插件都不能用了。包括ubiquity。后来只好找替代性的插件，又浪费了很多时间，于是自我总结，赶时髦并不是什么好事，至少像我这样的爱定制的用户是这样的。
  2. window的更新也很烦人。不断的有补丁，不断的需要打补丁，不断的有漏洞。操作系统吗，可不可以一劳永逸啊。而且有些补丁不能不打，否则程序就运行不正常，晕，程序设计哪有这样设计的，一定也没有长远眼光。有点羡慕linux了。
举例完毕。

谈谈感受：
现在的软件，好像总是再升级，升级是可以的，但是不能先把兼容问题解决好，再生吗。至少软件开发者并没有设身处地为用户着想过。其实新旧更替是避免 不了的，但是不能因为更新了，反而确用不了了。这和生物体的更新境界差远了，所以我认为，搞软件开发的应该学学生物学，了解了解生物体的新陈代谢是如何进行的。就比如人的身体，每天都要更新细胞，但是也没听说过哪个器官因为要更新细胞而停止工作，或者不兼容。再有，大脑的神经细胞的化学分子也要更替，但是也没有听说过，谁因为蛋白质的更替而造成了失忆。

由此可见，我们现在的信息技术，还远没有达到生命的境界，但是向着这个方向发展，应该是正确的方向。

如何在自己的网站上添加相册

发表于 2011-11-21 更新于 2013-01-11 分类于 flash

最近我们开展了一些学习活动，我想把活动的照片做成自动播放的相册进行展示。
这个问题如何解决呢？

方法一：利用网络相册flickr
上传图片到自己的flickr的某个图片集里，让后点击分享按钮，复制嵌入代码，粘贴到自己的网站页面上或者插件中。

方法二：利用网络自定义相册
有很多工具，可以做视频，效果也很不错。

方法三：自己做flash相册
有的时候用别人的平台和技术省事是省事，不过自定义的空间往往非常有限。刚巧前不久学了flash，可以自己动手做一个相册。见项目：ecnubio - ZCLflashxmlAlbum: http://ecnubio.wikispaces.com/ZCLflashxmlAlbum

总结：

  • 解决问题常常有多种方式方法
  • 运用代码帮助自己解决问题
  • programing for myself and share with friends
