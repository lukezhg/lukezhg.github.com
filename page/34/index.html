firefox常用扩展和技巧

发表于 2011-11-18 更新于 2013-01-11 分类于 Firefox ， web2.0

其实火狐浏览器有很多非常优秀的插件，如果使用火狐，但是没有定制和安装自己的插件，就好像养了一只孔雀，但是没有尾巴一样，错过了它最华丽的一面。当然插件如果安装的太多，且你的计算机配置又不高的话，可能会使得火狐启动缓慢。但是如果你一个插件也不定制，那真就是养了一只没有尾巴的孔雀了。

其实使用火狐浏览器的很多问题，都是通过安装扩展和插件解决的？下面罗列了一些我自己遇到的问题，以及解决的思路。详细的解决步骤以后再写。

如何设置gmail为firefox默认邮件客户端？

  • 打开火狐设置选项卡，选择应用，搜索mailto，修改为use gmail
如何去除firefox插件管理器里的灰色插件？
  • 找到并修改extensions.rdf文件，去除响应的<rdf>单元
  • 我个人的文件在：C:\Documents and Settings\luke\Application Data\Mozilla\Firefox\Profiles\wml2g9sr.default\extensions.rdf
如何让网页更具有可读性？
  • 安装扩展Readability，并设置浏览模式。
如何利用地址栏进行搜索？
  • 在地址栏输入about:config，回车，修改
  • 安装扩展Instant Fox，并自定义新的搜索。
如何自定义鼠标手势？
  • 安装扩展FireGestures，并自定义自己的鼠标手势。
如何定制特定页面的呈现方式？
  • 安装扩展Customize Your Web，并自定义特定页面的运行脚本语言。
  • Customize Your Web Feature Overview - YouTube
    : http://www.youtube.com/watch?v=gHOzLyNgldU&feature=player_embedded#!
如何截图并添加注释，然后发布？
  • 使用扩展FireShot
  • 现在diigo也可以做简单的截图和注释了，感谢diigo.
如何更方便的发博客？
  • 使用scribefire扩展
如何下载网页上的视频？
  • 使用Flash Video Downloader
如何增加网页浏览的面积？
  • 使用Hide Caption Titlebar Plus
  • Tree Style Tab
如何随时随地记录？
  • 使用diigo
  • list.it
  • QuickNote
  • Evernote
如何快速分享网页？
  • Shareaholic

什么是开放学习

发表于 2011-11-17 更新于 2013-01-11 分类于 openlearning

目前，网络上已经有很多学习资源了，人们称之为开放教育资源（OER）。
但是开放教育不仅限于资源的开放，还应包括教育过程的开放，以及学习过程的开放。因此除了开放课程，公开课，作为学习者，我们也应该开放的进行学习，并把自己的学习过程也开放，为更多的学习者提供线索。

那么什么是开放学习呢？

  • 开放学习是学习者依照自己的兴趣独立自主的学习。
显然，开放学习不仅需要学习者具有很强的自我调节能力，还需要一个支持性的外部学习环境。而这样的环境需要每个开放学习者的努力去营造，为此开放学习者要学会：
  1. 记录并分享整个学习目的、学习过程和学习成果
  2. 在学习过程中善于利用他人的学习经验以及集体智慧
  3. 有意识的创造和维护一个良好的学习环境

  • Open Learning Initiative: http://oli.web.cmu.edu/openlearning/

火狐扩展之ScribeFire让你轻松写博客

其实火狐浏览器有很多非常优秀的插件，如果使用火狐，但是没有定制和安装自己的插件，就好像养了一只孔雀，但是没有尾巴一样，错过了它最华丽的一面。其中ScribeFire就是孔雀尾巴上一只绚丽的羽毛。

如何把tiddlywiki笔记导入到evernote

发表于 2011-11-11 更新于 2013-01-11

tiddlywiki是一个很不错的非线性网页笔记本，它是一个存在你硬盘上的一个单个html文件，但是打开之后你会发现它实际上是一个用语言写的笔记本程序，而且你可以更具自己的需要进行界面以及插件的定制，比如我为学生定制的tiddlywiki(http://ecnubio.wikispaces.com/tiddlywiki)。

在tiddlywiki已经有了很多笔记之后，它的打开和保存速度会有所下降，而且我使用了很多笔记服务，我想搜索笔记的时候经常忘记，这个笔记是在博客上，还是在diigo上，还是在googlereader上，还是在tiddlywiki上，还是在evernote上。所以你看，我的问题很明显，我要要把所有的笔记最好能在同一个地方存储一份。（这个问题让我很头疼，若果有其他更好的解决方法，不妨赐教）

所以我要把tiddlywiki上的笔记导入到evernote以便备份和搜索。尝试的解决方案：

  1. 使用ExportTiddlersPlugin: (http://www.tiddlytools.com/#ExportTiddlersPlugin)导出，纯文本文件，然后再编写文本分割程序，将每个笔记存为单独的txt文件。然后使用evernote-工具-导入文件夹。但是遇到了问题，第一导出的纯文本非常混乱，而且中文出现了乱码。折腾了半天，这个方案弃用。
  2. 又重新搜索，于是找到了一个老外的解决方案（The Strange Zen Of JavaScript: TiddlyWiki hack: exporting tiddler HTML: http://jszen.blogspot.com/2005/06/tiddlywiki-hack-exporting-tiddler-html.html）。在这个方案里提到使用火狐浏览器，可以通过选择tiddlywiki中的所有文字，然后右键，参看所选源代码，果然我看到了tiddlywiki中的所有笔记，而且这个笔记的格式非常统一，于是就动手把相应的笔记拷贝到一个txt文件中。接下来使用py编写了一个简单的文本切割程序，把文件中每个笔记单独的保存到一个txt文件中，然后导入，大功告成。yeah！！
总结：
  1. 动手之前好好做一下搜索工作非常必要，还是那句话，你遇到的问题99%已经有人遇到了过了，参考使用别人的解决方案，可以避免无谓的时间浪费和重复性工作。
  2. 遇到不熟悉的领域，找到相关的论坛进行搜索和求助是更容易的事情。
  3. 学好英文是非常关键的，这决定了你的阅读范围。能看英文的文档能让你的阅读范围扩大很多倍。多少倍呢？个人保守估计10倍左右，这取决于互联[[网上不用语言资料所占的比例]]。

教师应该是一个善于讲故事的人

发表于 2011-10-29 更新于 2013-01-11

很小的时候，我就很喜欢听故事。等大了一点，还编一些非常离奇的故事，将给小孩子听，虽然是现编现讲，而且非常离谱，但是它们居然还听的津津有味。这说明不了我多美会讲故事，只是说明孩子确实是非常喜欢听故事的。为什么呢？我觉得故事最贴近我们的生活体验，它是按照我们的个人经验范式叙事的，同时它也可能刚好符合了我们大脑的神经网络的情景记忆模式。

作为老师，要把一堂课上好，要用好的例子，比喻，表征，还有就是要会讲故事。无论是自己亲身经历的故事，还是别人的故事。善于搞科研的老师讲科研的故事，善于实践和反思的老师讲自己的故事，广泛阅读的老师讲经典的具有教育意义的故事。故事可以让你的课堂生动起来，鲜活起来，丰富起来，有趣起来，成为一个整体。当然前提是，你要善于讲，会讲故事。

教师讲故事应该把握一些原则：

  1. 主题相符（符合教学需要）
  2. 真实生动（真实可鉴，形象具体，绘声绘色）
  3. 短小精悍（占用时间短，语言精练）
  4. 发人深思（具有冲击性和启发性）
  5. 智慧活泼（让人会意一笑）
为了运用好这些故事，教师要注意日常的素材的搜集、整理和加工，最后才能为我所用。比如今天，我就在新世界数字出版网上看到一本在线读物《18岁以前孩子应该懂得的做事方法》（http://www.nwp.cn/book/233_6871.shtml），上面运用了很多故事素材，来达到教育意义。

教师也应该是一个有故事的人。平常的教学反思、教育叙事、生活经历都可以积累成自己的故事。

帮助学生提出他们自己的问题-跨上思想的野马

发表于 2011-10-17 更新于 2013-01-11

提问实际上是一个思考的过程？而提问是一种尝试建立连接的过程，因此对于学习非常有益。这种益处表现在，一方面，当你提问的时候，你会觉得寻找问题的答案是自己的责任，你会想方设法解决它。而当学生知道如何提出自己的问题时，他们会为自己的学习负起更多的责任，深化交流，建立新的连接和获得新的发现。

教师借助问题生成技巧（Question Formulation Technique）来辅助学生提问。

  1. 教师设计问题的焦点（图片、视频、声音，获得学会生注意，使其快速提出问题）
  2. 学生提出问题（规则是尽可能多的提出问题，不要停下来讨论，评判或者试图回答，写下原封不动的写下这些问题，把每个表述都变成问句。一但学生工作起来，这个规则可以使得学生进入开放的思考状态。）
  3. 学生完善自己的问题。区分哪些是开放性的，哪些是封闭性的问题。它们的优缺点如何，教师可以让学生至少把一个开放性问题转变成封闭性问题，并且把一个封闭性问题转变成一个开放性问题。让学生意识到，如何表述一个问题，会影响问题的深度、质量以及获得信息的价值。
  4. 学生优选自己的问题。单元学习开始时，选择你最想扩展学习的问题。实验时选择三个可以检验的问题。
  5. 师生共同决定下一步行动。
  6. 学生反思自己学到了什么。
