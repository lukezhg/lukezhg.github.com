fighting with the problem in my coumputer

These days I am fighting with the Display Card Problem of my computer. Befor coming to Georgia, My coumputer can not start normaly, the screen is black, athough you can hear the sound of login. I try to ingnore this problem by using another OS Ubuntu in my comuputer. BUT, unfortianaly, I can’t use the PAWS wireless network of UGA in Ubuntu. So I have to solve the Display problem in WIN7 or the wireless connection problem in Ubuntu. Guess what, neither of the problem is easy to trackle.

Finally, I solved the Display Problem using following steps:

  • setup the two display driver software in my drivers folder(“display/amd & amd2”) in C disk.
  • setup the downloaded display controler(“E:\DRIVERS\chunleidownload\13-9_win7_win8_64_dd_ccc_whql.exe”), uncheck the quick stream function(20131104 I didn’t do this but did it in previsous steps, then I use black but swichable mode)
  • everything comes to normal although I still can’t use swticheable display mode in BIOS

最终的解决方案
首先我重新安装了drivers文件包下的所有驱动，并下载和安装了一个最新的controler。
其实这并不是问题的实质，因为电脑每次并不是不能启动，而只是会自动进入投影仪模式，按切换键后才能自动切换回来。
但是这究竟是什么问题导致的却不清楚，我尝试了很多方法：
1 重新安装驱动：不管用（一次我成功的用单显卡启动了一阵，但毕竟不是最佳的显示性能）
1 重装系统：这招肯定管用，但是太累了
1 后来有人提到说把显示器中的一个“隐藏该显示器无法显示的模式”设置勾掉就行了，结果我发现我的显示器设置中，这项是灰色，而且本来就没有勾选。
1 再后来有个哥们在笔记本休眠到一半的时候按了启动键，结果也遇到了跟我一样的问题，而且也尝试了很多办法，后来发现这个问题跟硬盘的虚拟文件有关。我的电脑只有C盘有虚拟文件，由于我的内存有6G，我就限制了一下虚拟文件的大小，这次我把它改为系统托管。问题就解决了。
2014年1月7日，这次着急关电脑，可能再睡眠的状态按了电源键，结果又出了同样的问题，尝试了多次，最后在虚拟内存设置选项卡中勾选“系统托管”全部硬盘，问题就解决了。

总结一下
遇到问题 还是要自己思考所有可能相关的细节 进而列出可能的尝试解决方法。
还要看看遇到同样问题的人是如果解决的，也可进行尝试。

这就像做研究一样：
最开始是很实际的需求，碰巧解决了，也不会寻根问底。
但是反复遇到这个问题，有的时候有效，有的时候无效，这就让人想彻底解决这个问题。
于是自己开始不断探索，不断查阅别人的探索经验，总结教训，寻找逻辑关系。
最好终于问题解决，缘由寻到。

解决问题的技巧：
显示主要到是什么事件引发了问题？（休眠，连接投影仪，更新，）
接下来问题的表现是什么？（启动时黑屏，但是可以听到windows正常启动的声音，并且可以通过显示模式变为正常）
如何解决这个问题（不解决并加以利用，反正你自己会用就行；自己尝试；看看别人的尝试；思考前因后果并提供更可能的假设）
总结、记录和分享。

如何写教案、如何说课

发表于 2013-07-29 更新于 2020-04-10 分类于 教学

经常有学生或教师问我该如何说课或者写教案。我听到后，第一反应是“查”。自己主动查一查，如何说课这个问题，肯定有很多资源的。在我看来，说和写都是表达教学设计的方式而已，关键还是教学设计。说其实是口头表达，其实如果你能写好教案，设计好教学，那么我相信你也能表达好。所以最关键的是理清自己的教学思路。自己不清晰，怎么能指望别人听懂呢，更别指望受到赞赏和认可了。

我觉得说课要做到以下几点：
一、说课的内容一定要包括：前端分析、教学目标、教学思路-问题线索-学习活动、最后是学习检测。第一部分相当于先了解一下脚下的“路况”如何，第二部分相当于你是要告诉别人，在这样的“路况”下，你要去哪，第三部分是你要如何走这条“路”，第四部分是你如何知道你离“目的地”还有多远。

所以，概括起来说课包含以下内容：
说情境：教学情境是怎样的？
说目标：目标是如何确立的？
说过程：教学活动如何开展？为什么这么上？
说评价：你如何在教学过程中判断学生达成目标的情况。（这部分，很多老师都会忽略）

二、说课形式可以是多样的，可以借助ppt，也可以借助概念图，我推荐使用概念图，因为它能让你兼顾部分和整体、学习目标与学习活动之间的联系以及学习活动的过度和衔接等。这些都是新手教师容易忽略的。

三、说课的境界

  • 第一个层次，是让人听懂，知道你要如何上这门课，这是最基本的层次。
  • 第二个层次，是让人不仅知道你怎么上，还知道你为什么这么上，教学是教师有意识的行动。
  • 第三个层次，是让人觉得你是在创造性的解决实际问题，并且认同你的做法的合理性和可行性，教学是基于研究和经验摸索的创新尝试。

四、说课好坏的评判标准：
第一，是教学目标设计和表述是否合理。
第二，前端分析是否扎实、全面，是否包括学情分析、教材内容分析以及学习需要分析，是否能依据分析确定合理的教学目标。
第三，教学思路是否清晰，问题线索是否自然，学习活动是否能帮助学生达成教学目标。
第四，是否设计了新问题情境检测学生的学习成果和教学目标达成情况。

保持积极主动，坚持总结回顾

发表于 2013-01-10 更新于 2013-01-13 分类于 日志

对于一个学习者而言，如若想进步，我觉得必须要做到两条：

  1. 保持积极主动
  2. 坚持总结回顾

这两条听起来好像很简单，其实很难做到。

比如积极主动，我曾经鼓励我的学生主动联系附近中学的一线老师，进而寻求合作。也曾经鼓励学生为了寻找一篇文献给名人写信。但是我发现最后学生们很少有真的行动的，学生还没有养成积极主动的习惯。尽管他们已经是研究生了，尽管他们接受了16年的教育，尽管他们即将面临写论文、就业，但是他们仍然没有养成积极主动的习惯。这不禁让人反思我们的教育所起的作用。

积极主动除了行动之外，还可以表现在思考上。在工作或学习过程中，你是否愿意信任伙伴，你是否倾向于把别人往好处想，你是否处于过度防范的状态，对这些问题的回答就能反映你是否会积极的思考。积极思考的人往往也能感染别人积极的思考，相信别人的人，也更容易取得别人的相信，进而建立积极的人际关系和工作氛围。人家都说中国是关系社会，社会工作关系复杂，让人很累，每个人都处于防范之中，能不累吗。其实很重要的一点就是没有养成积极思维的习惯，遇到事情总是朝着猜忌和负面去考虑。所及积极主动不仅对自己很重要，对于环境的影响也不可忽视。

那么回顾中应该很容易做到了吧，其实不然。

人是需要回顾过去的，比如回顾一年的工作和生活，回顾一个月的收获和行动，回顾一个星期的时间花费。等等。但是有的时候我们实在太忙了，忙的没有时间回顾，没有时间思考，没有时间想一想我们到底在忙些什么。

我曾经这样告诫自己，要立长志，不要常立志。但是很多时候还是做不到，还是常立志，后来想想常立志也还过得去，只要每次立的都是同样的志向，那么也就在接近立长志了。

我有一个坏毛病，容易虎头蛇尾。可能这也证明我自己其实地的确确就是一个普通人，坚持做一件事情是困难的，坚持把一件事情做好更是难上加难，但无论怎样，我还是要我劝慰，be proactive, 积极的思考和行动。很多时候，失败就失败在不积极、不主动、充满了猜疑和戒备，那样的话成功的几率几乎为零。

因此，我不禁要劝自己积极的思考和行动，也要去鼓励我周边的人积极的思考和行动，用正能量去感染自己和他人，让我们共同更接近成功！

宝宝跳跳器——脚手架

发表于 2012-04-26 更新于 2013-02-18 分类于 生活



“跳跳器是一种悬挂在门框金属夹具上的塑料椅或布椅。它由链条、弹簧或橡胶和布带构成。宝宝在上面跳起，就可以弹离地面。一些宝宝不喜欢跳跳器，而其他宝宝却很喜爱，直到学会走路。

跳 跳器也有一些问题：夹具可能会松开，带子可能会磨断，从而导致宝宝掉到地上或撞到门框。一些座椅对宝宝的大腿内侧压力过大，导致瘀痕或影响血液循环。安装 跳跳器也具有一定的难度。宝宝长时间使用会导致晕眩，症状与晕船相似。同时还有鞭抽式损伤的危险，因此我们不提倡使用跳跳器。” 摘自：怎样为刚刚学走路的孩子选择用具？ 博闻网: http://health.bowenwang.com.cn/how-to-choose-baby-equipment8.htm

今天偶然在路上可到有一个家庭正在给宝宝使用跳跳器，这让我联想到脚手架的原理。宝宝在还没有学会走路的时候，家长有时候会用双手架在宝宝的腋下帮助其承担一部分的重量，这样宝宝就可以在双腿力量上不足够支撑身体重量的时候学习走路，这更我们的脚手架原理是一样的，学习者在教师提供特定帮助下完成学习任务，然后这种帮助会慢慢减少，直到学习者可以独立完成任务。

这正应了那句名言“教是为了不教”。
还有一句是“不教也是一种教”。

实验教学理论课的教学反思

发表于 2012-02-20 更新于 2015-03-10 分类于 教学

今天上了实验教学的理论课，一个很深的体会是如果让学生连续听2个半小时的讲座，想让学生一直高度关注是很难的。反过来，让一个教室连续讲上2个半小时，想不累也是很难的。今天这次课就是同时做了这两件事情。

成功之处：
觉得一些好的例子，是很不错的。比如小学探究热量的案例，光合作用影响因素的案例。此外三盒米饭的例子也很不错。对于思想的形象化解释和比喻也是不错的。在讲解的过程中调动学生的已学知识和概念的做法也是好的。

不足之处：
总是想让学生动起来，不仅仅是行动起来，同时也是思考起来，活跃起来，更愿意表达核分享，但是同学还是不是十分想敢说。有啥说啥的勇气还是缺乏。
另外想讲的内容很多，而且ppt没有大标题标记，显得有些混乱，标题和标题之间的线索有些不清晰了。z老师提出建议其实可以把所有内容概括为三大块，做好大标题标记，并去掉一些多余的案例和干巴巴的理论，会更好。这些建议是中肯的。
在语言表的基本功上，我的语言表达能力还需要练习，表达得特别快的时候就会找不到合适的词语或者吐字不清，这点其实要在平时多积累词汇，多写，多说，厚积薄发，自然水到渠成。

突发事件：
刚开始想使用投影仪，但是发现怎么弄都投不上去，于是换了一个接口，头上去了。
讲授的时候发现投影仪接口与前面的讲台很远，于是自己搬了个凳子在加了一个箱子做了一个临时讲台，效果不错。这样可以看电脑上的ppt也可以看到同学的眼睛了。（但是还是应该考虑买一个ppt遥控器）
本来想使用ofiice2007的演讲者播放模式，这样可以看自己的笔记同时大屏幕只显示演示内容，但是弄了一些没成功。
分组的时候，有两个组一个组全是女生，一个组全是男生，为了男女搭配，我只好让他们重新组合一下，但是有一点强制性了，对不住这两位女同学了，把她们调过去的时候她们可能不是特别乐意。现在想来还是可以让他们私下商量解决的，不必要教师干预。毕竟他们都是成人了。

学生见解：
有同学提出想借用一楼的微格教室，但是为了管理方便，还是建议他们使用实验教室。但是提前就多练习和操练微格的这个意识是值得充分肯定的，这也说明有些同学确实是非常主动的。

快速发布博文的方法与技巧

发表于 2011-12-17 更新于 2013-08-25 分类于 技术技巧

发布博客的方法大体上可以分为三类：

  1. 利用本地软件发布博客。比如使用windows live writer等安装在本地电脑上的软件来发博客。这种方法在撰写长博客的时候会非常稳定，不会受到网络的干扰，即便突然断线或者离线，也可以照写不误，同时留有本地备份。但是缺点是并不是所有博客都支持离线发布，而且这个软件也势必要占用磁盘空间，再有发布的时候，如果需要参考网页信息的话，需要来回切换窗口，比较麻烦。
  2. 利用网络浏览器的插件进行博客发布。比如火狐浏览器的scribefire以及谷歌浏览器的嘉写等。它的优点是你不需要离开网络浏览器，只需要使用快捷键启动插件，就可以发布。同时你可以很方便的引用网页上的文字、网址等信息。尤其是scribefire，它可以分屏显示，上面是特定的网页，引起了你的思考，你就可以按F8快捷键调用该插件，并在下方的编辑区中编写你的博客内容，然后发布。这种方法的好处是不需要再多安装一个软件，而仅仅是对你的浏览器进行一下扩展，再有它也会自动留备份在你的本地电脑，连线的时候也可以写博客，等有了网络再发布。
  3. 在线编辑博客。这个方法不用多说，大多数用回都用这正方法，缺点对网络依赖非常强，若果一旦不稳定或者浏览器崩溃，你的工作就可能白费。优点是最少的占用你的本地资源，而且所有的可能功能都容易实现。其实使用的时候可以利用一些技巧，比如为你写博客的网址建立一个书签或者鼠标手势，并保存密码登录（限个人电脑），这样你只要点击一下书签或者画一个手势，就自动进入编辑页面了，效率大大提高。

用flash制作试题交互动画项目

发表于 2011-12-17 更新于 2013-01-11 分类于 技术技巧 ， flash

在网络中有很多互动的练习题都是用flash制作的，互动效果很好，学生可以利用这些动画巩固学习的知识，检测自己的掌握程度。但是很多练习题都是不提供源文件的，即便提供了原文件往往试题数据和源文件并没有分离，教师很难更改并为我所用。那么是不是可以创建一个flash源文件，让稍微动flash甚至是一点都不同flash的老师也能修改并展示自己的试题呢？当然是可以的，但是这个试题交互动画的设计应该数据与源文件分离的，也就是老师不需要修改源文件和代码，只需要修改数据文件就可以。

制作的思路：
1、建立界面fla原文件（test.fla）
2、建立代码文件(xmlloader.as)
3、建立试题数据xml文件(sample.xml)

使用代码加载试题数据xml文件，并在动态文本中展示试题，学习者点击选项按钮以及导航按钮参与互动。互动可以分为两种模式，一种是练习模式，一种是测验模式，用户可以自己选择。此外为了使得该动画可以在网站上调用，还增加一个输入文本框，用于导入特定位置的试题数据。

发布：
最有作品提供全部文件，并以CC协议中的（署名 非商业使用 相同方式共享） 发布。

低效率从失控开始

发表于 2011-12-16 更新于 2013-01-11 分类于 反思

不久前曾经看过一本书，叫《out of control》，中文名叫失控，不过只看了开头。书中介绍人类设计的系统最终会走向失控。

其实，不仅仅是人类设计的机器会失控。其实人本身又何尝不是。总是有那么几次，自己很难控制自己去做某些重要的事情，而偏偏做了一些并不重要的无关的事情，这就是失控。失控是低效率的罪魁祸首，同时好像又避免不了，至少对于我而言是这样的。每次忙完一周之后，原本打算干点正事，但是总是被一些电脑的、网络技术的一些并不太重要问题和事情纠缠掉所有的时间。而重要的事情，则再次落空。长此以往，很难成功。

从这些失败的教训让我反思，失控是一种常态，我们能做的是控制失控的程度，不至于让它产生过于严重的危害。比如避免成瘾，避免危险，避免死循环等等，恰当的学会失控，顺其自然也学比一味的约束自己有用。

为什么会失控呢，这是本能以及周遭的环境再作祟。

基于问题的学习和知识分享（PBL &　PBS）

发表于 2011-12-01 更新于 2013-01-11 分类于 openlearning

我们在教学和学习的过程中，经常会使用到基于问题的学习（problem-based Learning）策略。原因很简单，因为人们经常是为了现在解决遇到的问题而学习的，而不是为了解决未来的问题而做准备，因为问题是很难预知的。遇到了，才知道。

在基于问题的学习中，学习者可以基于自己的兴趣扩展学习，去查阅资料，却做尝试和实验，此外还要肩负起监控和调整自己学习过程的重担，元认知也能够得到锻炼。如果不能达到这些，这种教学策略对于学习成效的提高就很难起到帮助。

我们不仅可以围绕问题展开学习，同样的我们也应该围绕问题展开知识的分享。在围绕问题分享知识的时候，我们要做三件事情：

  1. 具体清晰的描述自己遇到问题和自己想达成的目标。
  2. 给出问题解决的关键思想和具体解决步骤。
  3. 提供相关的扩展阅读的资料和参考文献。
学习者完全可以只用这三个环节来分享知识，这会使得知识的交流的焦点更为集中，分享更有针对性和成效性。通常我们经常缺乏清晰描述自己问题的意识，以至于自己的解决方案究竟在遇到什么问题时才会派上用场也不清楚。还有就是只关注了给出具体的步骤，但是并没有提炼问题解决的核心思想，这样容易让读者只有步骤细节，而缺乏整体观，很容易产生误解和迷惑。此外，提供信息的参考来源有利于让读者了解信息的前前后后，或者为了进一步扩展阅读提供了可能。

会心跳的空调

发表于 2011-11-30 更新于 2013-08-25 分类于 生活

今天天气不好，一直下着小雨，雨不是很大，但是借着较强的风势，横飞乱舞，打着伞身子也会被淋湿了一半。回到家，坐在办公桌前面打开电脑，正准备做一个flash演示文稿模板，突然听到一种很奇怪的声音，嘭~啪~嘭~啪~，周而复始，有点像流水的声音，又好像似一个人的心跳，但是家里明明只有我一个人。

于是仗着胆子站起身来，细听声音的来处，原来就是侧对着办公桌的哪个空调发出的声音。好奇怪啊，这个空调明明没有开，怎么会有声音呢。外面的风刮的更厉害了，又传来了嘭啪嘭啪的声音，于是猜想是不是与这风有关系呢？本来想静下来做点事情的，这个空调老是给我执照噪音怎么行呢，于是就开始想解决的办法。

我猜想可能是风速变化很大，时快时慢，使得空调的循环管受的压力不断变化，进而造成里面的液体的震荡所以产生了这种类似“心跳”的声音。加入这种猜想成立的话，如何解决呢? 首先想到的是用一个大塑料袋子把空调的风扇整个包起来来，这样这个风扇的管道就不对外开放了，但是找了一会没找到合适的袋子，而且风扇在阳台栏杆的外面，需要越过栏杆才能将其罩住，栏杆很脏，而且有一定危险性，于是放弃了这种方法。那么怎么办呢？

仔细想了一下，由于天气有点冷，房子的门窗都关着，整个房子就好像一个秘密的容器，只有空调的循环管与外面相连，由于风速变化造成压力变化，所以会传来空调的心跳声。对了，如果我在打开一个通道不久行了嘛，于是我就把阳台的拉门打开了，果然空调的心跳停下来了，问题解决！

什么是知识共享协议（CC协议）

发表于 2011-11-28 更新于 2013-01-11 分类于 openlearning ， web2.0

在网络学习的过程中，我们经常会使用用别人的作品，或者我们自己也会制作一些作品，那么那些作品可以进行演绎创作，哪些不可以，我们使用了是否侵权，我们制作的作品又再什么样的协议下发行，这些成为迫切需要解决的问题。

目前网络上使用广泛的，大家一般共同遵循的协议就是CC协议，英文为（Creative Commons），目前大陆也有网站叫做知识共享中国大陆。

在版权声明中经常会使用一些图标，比如署名、非商业性使用、禁止演绎以及相同方式共享等图标。使用者可以组合这些图标来声明自己作品的许可协议。那么这些图标的含义是什么呢？这里摘录知识共享中国大陆网站的一段文字如下：

许可协议说明

您在自己的作品上使用知识共享许可协议，并不意味着放弃您的著作权，而是在特定的条件下将您的部分权利授予公共领域内的使用者。哪些特定的条件呢？您可以在此处看到所有知识共享许可协议及其简单的介绍。所有的许可协议都要求您以作者或者许可人的名义署名。

您可以将以下的选项进行组合、搭配，由此将构成我们的六套核心知识共享许可协议。

大脑错觉（Brain bugs）

发表于 2011-11-27 更新于 2013-08-25 分类于 book

在笑来的博客上看到，这本书《Brain bugs》，笑说有趣的很。于是打开看了一下目录还有网站的Demos如下：
Brain Bugs | Demos
: http://www.brainbugs.org/demos.html

上面的页面上列举了很多brain bugs。
比如McGurk Effect，上面有一个女人在发言，但是当你闭上眼睛的时候，你听到的是一个声音，当你睁开眼睛的时候你听到的是另外一个声音。（我猜测是收到了发音者口型的影响）

再比如Baker-baker Paradox
我们经常会忘记某个人的名字，但是多半会记得他的职业。即使是一个叫Baker的baker，我们还是容易自己的他是一个面包师。原因在于，我们记忆一个人的职业的时候，会有很多相关的概念协同起来，而名字则很少协同记忆。（也许老师们也可以尝试为学生的名字富于内涵，从而便于记忆）
这也提示我们，在教学过程中，应该把新概念与学生的已有概念产生更多的来联系，使其协同起来，而不是像名字一样孤零零的，最后被遗忘。

再如Flash Lag Effect
我在预测红球的时候，总是预测超前50左右，这就是为什么足球边裁判罚越位中有24%为误判的原因。其原因是我们的视觉系统在关注其他信号的同时，总是倾向于超前估计运动物体的位置。

再如Leaning Tower Illusion
看起来右侧的塔会更加的倾斜，但是其实两个塔的照片是完全相同。

从软件更新到新陈代谢

发表于 2011-11-27 更新于 2013-01-11 分类于 Firefox

最近有点反感软件更新了。为啥呢？主要是兼容问题。
举例：

  1. 我以前用火狐浏览器挺好的，安装了很多好用的插件，也花了好多时间和心思定制自己的火狐。有一天，看到了火狐的更新提醒，于是点击了更新，结果原有的很多插件都不能用了。包括ubiquity。后来只好找替代性的插件，又浪费了很多时间，于是自我总结，赶时髦并不是什么好事，至少像我这样的爱定制的用户是这样的。
  2. window的更新也很烦人。不断的有补丁，不断的需要打补丁，不断的有漏洞。操作系统吗，可不可以一劳永逸啊。而且有些补丁不能不打，否则程序就运行不正常，晕，程序设计哪有这样设计的，一定也没有长远眼光。有点羡慕linux了。
举例完毕。

谈谈感受：
现在的软件，好像总是再升级，升级是可以的，但是不能先把兼容问题解决好，再生吗。至少软件开发者并没有设身处地为用户着想过。其实新旧更替是避免 不了的，但是不能因为更新了，反而确用不了了。这和生物体的更新境界差远了，所以我认为，搞软件开发的应该学学生物学，了解了解生物体的新陈代谢是如何进行的。就比如人的身体，每天都要更新细胞，但是也没听说过哪个器官因为要更新细胞而停止工作，或者不兼容。再有，大脑的神经细胞的化学分子也要更替，但是也没有听说过，谁因为蛋白质的更替而造成了失忆。

由此可见，我们现在的信息技术，还远没有达到生命的境界，但是向着这个方向发展，应该是正确的方向。
